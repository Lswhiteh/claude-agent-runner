# Plan: Add Claude Code Hooks Suite to claude-agent-runner

## Context

The agent runner currently has only 2 hooks (block-destructive.sh, scope-guard.sh) — both PreToolUse:Bash guards. Claude Code supports 16 hook event types with rich control (blocking, context injection, input rewriting, async execution). This plan adds 9 new hooks across 7 event types to improve agent safety, observability, and self-correction. It also fixes the broken settings.json registration format in setup.sh.

## New Hooks (9 files)

### 1. `hooks/stop-gate.sh` — Stop event
- Gate: `CLAUDE_AGENT_MODE=1`
- Checks `stop_hook_active` to prevent infinite loops
- Parses transcript JSONL for evidence of CI/test runs (ci-gate, npm test, pytest, cargo test, vitest, jest, go test)
- If no test evidence found → `{"decision": "block", "reason": "Run CI checks before finishing"}`
- Falls back gracefully (allows stop) if transcript unreadable

### 2. `hooks/rtk-failure-hint.sh` — PostToolUseFailure:Bash
- No gate (useful everywhere, not just agent mode)
- When `tool_input.command` starts with `rtk ` but not `rtk proxy` → inject additionalContext: "Re-run with `rtk proxy <cmd>` to see full output"

### 3. `hooks/session-context.sh` — SessionStart
- Gate: `CLAUDE_AGENT_MODE=1`
- Reads `CLAUDE_AGENT_ISSUE_ID`, `CLAUDE_AGENT_REPO`, `CLAUDE_AGENT_BRANCH`, `CLAUDE_AGENT_WORKTREE` env vars
- Outputs additionalContext with agent identity summary
- Writes exports to `$CLAUDE_ENV_FILE` if available (persists env vars to all Bash calls in the session)

### 4. `hooks/auto-lint.sh` — PostToolUse:Edit/Write (async)
- Gate: `CLAUDE_AGENT_MODE=1`
- Extracts `file_path` from tool_input, checks extension
- Detects lint config in cwd (eslint for ts/js, ruff for py)
- Runs linter; output delivered as context on next turn
- Async: non-blocking, won't slow down writes

### 5. `hooks/command-budget.sh` — PreToolUse:Bash
- Gate: `CLAUDE_AGENT_MODE=1`
- Counter file at `/tmp/claude-agent-budget-${SESSION_ID}` (from stdin JSON)
- Increments on each Bash call
- At threshold (default `$CLAUDE_AGENT_CMD_BUDGET` or 300) → deny with "agent may be stuck" message

### 6. `hooks/subagent-collector.sh` — SubagentStop
- Gate: `CLAUDE_AGENT_SCOPED=1` (orchestrated workers only)
- Appends `{timestamp, agent_id, agent_type, summary}` to `.claude/orchestrator/subagent-log.jsonl`
- Pure logging, no blocking

### 7. `hooks/task-validator.sh` — TaskCompleted
- Gate: `CLAUDE_AGENT_MODE=1`
- Same transcript parsing as stop-gate (shared inline function)
- Exit 2 + stderr message to block task completion if no test evidence

### 8. `hooks/command-rewriter.sh` — PreToolUse:Bash
- Gate: `CLAUDE_AGENT_MODE=1`
- Rewrites `npm install <packages>` (without --save-exact) → adds `--save-exact`
- Uses `updatedInput` in hookSpecificOutput to modify the command
- Only fires for package installs, not bare `npm install` or `npm ci`

### 9. `hooks/pre-compact.sh` — PreCompact
- Gate: `CLAUDE_AGENT_MODE=1`
- Saves snapshot to `.claude/agent-state/compact-snapshot-$TIMESTAMP.md`
- Includes: git diff --stat, git status --short, git log --oneline -10
- Non-blocking (PreCompact cannot prevent compaction)

### NOT implementing: WorktreeCreate/Remove
Agent runner creates worktrees via `git worktree add` directly — these hooks only fire for Claude Code's built-in worktree feature, so they're not applicable.
